# Claude 하네스 정합성 리포트

이 파일은 `claude-code-best-practice`와 현재 Codex 번역 하네스의 핵심 운영 규칙을 비교한 결과를 기록합니다.

## 최근 점검

- 점검 일시: 2026-04-04
- 기준 Claude 저장소 커밋: 로컬 작업본 기준 수동 비교
- 결과: 대체로 일치. 일부는 Codex 특화 확장으로 `의도적 차이`

## 비교 결과

| 영역 | 결과 | 메모 |
|---|---|---|
| 브랜치 운영 규칙 | 일치 | `main`을 영어 기준 브랜치, `ko`를 번역 브랜치로 유지하는 2-브랜치 전략이 같다 |
| upstream 동기화 절차 | 일치 | `upstream` fetch, `main` fast-forward, `ko` 반영, 충돌 시 upstream 우선 수용 후 재번역 원칙이 같다 |
| `origin/main` 최신 유지 | 의도적 차이 | Claude 쪽에도 같은 취지가 있으나 Codex 하네스가 이 부분을 더 명시적으로 강제한다 |
| 번역 규칙 | 대체로 일치 | 식별자, 경로, 코드, 설정 키는 영어 유지하고 산문만 번역하는 원칙이 같다 |
| 용어/구조 보존 | 일치 | 기술 식별자를 보존하고 구조를 바꾸지 않는 안전 규칙이 같다 |
| 번역 대상 판정 방식 | 의도적 차이 | Claude는 화이트리스트/배치 중심, Codex는 규칙 기반 점검 + upstream 변경 감지 중심 |
| 검증 규칙 | 대체로 일치 | 구조 검증, 역번역 검증, 실패 파일 재검증 루프라는 핵심 의도는 같다 |
| 재검증 루프 | 의도적 차이 | Codex 하네스는 실패 파일만 재검증하는 범위 축소 규칙을 더 명시적으로 둔다 |
| 커밋 규칙 | 대체로 일치 | 한글 커밋 메시지, 브랜치 보호, 민감 파일 차단, 검증 체크리스트라는 큰 틀은 같다 |
| attribution 정책 | 의도적 차이 | Claude 저장소는 `Co-Authored-By`를 강하게 요구하지만 Codex 하네스는 현재 채택하지 않았다 |
| 하네스 문서 분리 | 일치 | 운영 문서를 번역본과 분리해 별도 경로에 두는 철학이 같다 |
| 상태 기록 규칙 | 일치 | 진행상황, 재번역 목록, 검증 결과를 `reports/`에 남기는 방식이 같다 |
| Claude 하네스 교차 점검 | 의도적 차이 | Codex 쪽에만 추가된 보강 규칙으로, Claude와의 정합성을 유지하기 위한 상위 안전망이다 |

- Claude 저장소의 하네스 규칙이 바뀌면 이 리포트를 갱신합니다.
- `attribution` 정책은 필요 시 별도 결정합니다.
