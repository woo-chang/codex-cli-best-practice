# 커밋 정책

이 문서는 번역 하네스 관점의 커밋 정책을 설명합니다. 실제 실행 규칙은 `.agents/skills/commit-rules/SKILL.md`를 기준으로 합니다.

## 기본 원칙

- 커밋 메시지는 `영어 접두사 + 한글 요약` 형식을 사용합니다.
- 번역 작업은 `ko` 브랜치에서만 커밋합니다.
- `main`은 upstream 동기화 전용 브랜치로 유지합니다.
- upstream 동기화, 번역, 검증 수정은 가능한 한 서로 다른 커밋으로 분리합니다.

## 권장 패턴

### 하네스 구조 추가

```text
feat: 한국어 번역 하네스 구조 추가
```

### 번역 배치

```text
translate: README와 AGENTS 번역
```

### upstream 동기화

```text
sync: upstream 변경사항 반영 및 재번역 목록 갱신
```

### 검증 반영

```text
verify: 역번역 검증 결과 반영
```

## 분리 이유

- upstream 변경과 번역 변경을 분리하면 충돌 원인을 추적하기 쉽습니다.
- 번역 후 수정과 검증 후 수정을 나누면 의미 변경 시점을 추적하기 쉽습니다.
- 배치 단위 커밋은 재번역 범위를 관리하기 쉽습니다.
