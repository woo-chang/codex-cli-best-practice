# 검증 규칙

배치를 번역한 뒤 이 체크리스트로 검증합니다.

## 엄격 모드 원칙

역번역 검증은 기본적으로 엄격 모드로 수행합니다.

- 문서 전체가 아니라 섹션 단위로 비교합니다
- 제목, 표, 불릿, 주의문, 지시문을 각각 비교 대상으로 봅니다
- `must`, `only`, `do not`, `required`, `default` 같은 제약 표현이 약화됐는지 따로 확인합니다
- 기술 용어와 시스템 식별자는 보존 여부를 별도로 확인합니다
- PASS 판정은 "핵심 의미 유지 + 제약 유지 + 식별자 보존"이 모두 충족될 때만 가능합니다

## 구조 검증

- 프론트매터가 유효한 YAML로 유지되는지 확인합니다
- TOML, JSON, YAML 키가 변경되지 않았는지 확인합니다
- 코드 펜스가 정상적으로 열리고 닫히는지 확인합니다
- 상대 링크가 기존 대상 파일을 계속 가리키는지 확인합니다
- 제목이 번역된 뒤에도 프래그먼트 링크가 계속 해석되는지 확인합니다
- 배지 URL과 이미지 경로가 변경되지 않았는지 확인합니다

## 번역 경계 검증

- 의도된 경우가 아니라면 코드 펜스 내부에 한글이 들어가지 않았는지 확인합니다
- 파일 경로, 명령어, 플래그, 식별자가 영어로 유지되는지 확인합니다
- 표의 열 구조가 원본과 동일한지 확인합니다
- 마크다운 포매팅 마커가 보존됐는지 확인합니다

## 의미 검증

- 번역된 지시가 원문의 의미를 유지하는지 확인합니다
- 사용자 대면 설명이 자연스러운 한국어인지 확인합니다
- 중요한 주의사항과 제약이 약화되거나 뒤집히지 않았는지 확인합니다

## 심각도 기준

- `ERROR`: 의미 반전, 핵심 정보 누락/추가, 제약 약화, 기본값 왜곡, 식별자 손상
- `WARNING`: 의미는 유지되지만 직접성, 뉘앙스, 용어 일관성이 흔들리는 경우
- `PASS`: 의미, 제약, 식별자가 모두 유지되는 경우

## 역번역 검토 항목

파일별로 최소 다음 항목을 기록합니다.

- 비교한 섹션 수
- 비교한 표 또는 목록 수
- ERROR 개수
- WARNING 개수
- 대표 차이 또는 "이상 없음"

## 기록

`reports/translation-progress.md`에서 배치를 완료로 표시하기 전에, 검증 결과를 `reports/back-translation-report.md` 또는 별도 검토 노트에 기록합니다. 엄격 모드에서는 가능하면 파일별 상세 섹션을 함께 남깁니다.
